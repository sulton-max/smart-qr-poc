# v0.5 — Code styling and routing

*Last updated: 2026-06-24*

**Status:** 🚧 In Progress · **Type:** Feature · **Started:** 2026-06-24 · **Completed:** —

### Iteration 1 — Server-side rendering

- [ ] Ability to render a code from one server-authoritative source whose builder preview matches the downloaded asset exactly.

### Iteration 2 — Code styling

- [ ] Ability to style a code with module and finder-eye shapes, foreground and background colors with gradients, transparency, a center logo, and a frame with a caption.

### Iteration 3 — Routing rules

- [ ] Ability to route one code to different destinations by device, country, language, and time, with a builder for the rules.

### Iteration 4 — Geo resolution

- [ ] Ability to resolve a scanner's country from their IP so country and language routing work.

### Iteration 5 — Verification

- [ ] Build both hosts + unit + E2E → all green.
- [ ] Create a styled code in the builder → preview matches the downloaded SVG and PNG byte-for-byte.
- [ ] Download a code as SVG and as PNG at 300+ DPI → both render the chosen style and scan.
- [ ] Add device + country + language + time rules → a simulated scan picks the right destination, falling back when none match.
- [ ] Scan from a foreign IP → the country resolves and the matching rule wins.
