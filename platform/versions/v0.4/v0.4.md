# v0.4 — SDK adoption (backend-beta + frontend-beta)

*Last updated: 2026-06-22*

**Status:** ✅ Complete · **Started:** 2026-06-19 · **Completed:** 2026-06-23 *(Iter 1–9: 2026-06-22 · Iter 10: 2026-06-23)*

> The adoption pass: run smart-qr on the wow-two SDK for everything that isn't business logic, and adopt `@wow-two-beta/ui` fully. **Adopt-first** — consume what the SDK already ships; SDK gaps get added to the SDK source (publish round-trip), never built app-local for keeps.

## Decisions (resolved)

- **D1 — error model:** keep `AppResult` + `FailureCategory`; the errors/validation/mapping redesign is a separate future chat, out of v0.4.
- **D2 — casing:** snake_case in the DB only (tables/columns/enum values); the API↔frontend JSON wire stays camelCase.
- **D3 — testing package:** ship `WoW2.Sdk.Backend.Beta.Testing` as a separate package (xUnit/Testcontainers/Respawn out of the runtime SDK).
- **D4 — frontend:** no `frontend-common` extract — replace with `@wow-two-beta/ui` in place; app-local infra (api client, `usePageMeta`, identity gate) stays app-local.
- **D5 — close SDK gaps in-version:** adopt what's published now; when the SDK lacks something, add it to the SDK source → owner publishes → app bumps and adopts. No deferral, just a publish round-trip.
