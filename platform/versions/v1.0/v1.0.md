# v1.0 — Manage codes (guest-first)

*Last updated: 2026-06-12*

**Status:** 🚧 Built — pending live verification · **Started:** 2026-06-11 · **Completed:** —

> Guest-first: codes belong to their anonymous creator via an owner key; sign-in is added later as a
> claim flow (backlog), so auth layers on rather than rewriting. Timebox ≤ 1 week.

### Iteration 1 — Manage codes

**Goal:** Creators can edit and organise their own codes, and an edit takes effect on the next scan.

- [x] A new code belongs to its creator (anonymous, no sign-in) so only they can manage it.
- [x] Browse and search your own codes; open one.
- [x] Change a code's destination and rules after it's printed (slug stays the same).
- [x] Enable, disable, or remove a code.
- [x] An edit reaches the redirect so the next scan resolves to the new destination.

### Iteration 2 — Management UI

**Goal:** A screen to manage your codes end-to-end.

- [x] See your codes in a list with their status and destination.
- [x] Edit a code from the builder UI.
- [x] Toggle a code on or off, and delete it.
