# v1.0 — Essential / Launch

*Last updated: 2026-06-09*

**Status:** ⏳ Planned · **Started:** — · **Completed:** —

### Iteration 1 — Manage codes

**Goal:** Owners can edit and organise their codes after creation.

- [ ] Change a code's destination and rules after it's printed.
- [ ] Enable, disable, or remove a code.
- [ ] Browse and search all of an owner's codes.

### Iteration 2 — Accounts

**Goal:** People sign in and own their codes.

- [ ] Sign up and log in.
- [ ] Each code belongs to its owner; owners see only their own.

### Iteration 3 — Scan insights

**Goal:** Owners can see how their codes are scanned.

- [ ] Show scans over time and by device, location, and OS.
- [ ] Tell unique scanners apart from total scans.

### Iteration 4 — Paid plans

**Goal:** Customers can pay within enforced limits and never lose a code.

- [ ] Subscribe to a plan and manage billing.
- [ ] Enforce per-plan limits.
- [ ] Codes keep resolving even when a plan lapses or downgrades.

### Iteration 5 — Production readiness

**Goal:** Run reliably in production on the real hot-path store.

- [ ] Evolve the schema safely between releases.
- [ ] Serve redirects from a fast shared cache that stays in sync with edits.
- [ ] Run live behind a CDN with TLS.

### Iteration 6 — Trust & safety

**Goal:** Keep the redirect safe to operate.

- [ ] Screen destinations for malicious links.
- [ ] Throttle abusive traffic.
- [ ] Accept reports and take codes down.

### Iteration 7 — Brandable codes

**Goal:** Make a code feel like the customer's own.

- [ ] Serve codes from a customer's own domain.
- [ ] Place a logo in the centre of a code.
- [ ] Gate a code behind a password.
