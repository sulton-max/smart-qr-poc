# v0.7 — Static content and export

*Last updated: 2026-06-30*

**Status:** ⏳ Planned · **Type:** Feature · **Started:** — · **Completed:** —

### Iteration 1 — Static content types

- [ ] Ability to create a code from a static content type — URL, contact card (vCard), WiFi network, calendar event, location, email, SMS, and phone, plus plain text — each with its own builder form.

### Iteration 2 — Barcodes

- [ ] Ability to create non-QR symbologies (Code 128, EAN, UPC, Data Matrix, PDF417, Aztec) from the builder.

### Iteration 3 — Export and print

- [ ] Ability to download a code as an SVG or PNG file instead of opening it in a browser tab.
- [ ] Ability to get a print-ready code — vector output with a sufficient quiet zone, error-correction headroom, and sizing guidance — that stays valid as later decorations (logo, frames) are added.

### Iteration 4 — Builder correctness

- [ ] Ability to see clear validation and a guaranteed-unique short link when creating or editing a code.
- [ ] Ability to learn what each builder control does from an info popover by its header.
- [ ] Ability to preview the exact server-rendered code in the builder so it matches the downloaded asset.

### Iteration 5 — Verification

- [ ] Create a code from each content type → the scanned payload is correct (URL, vCard, WiFi, calendar, geo, email, SMS, phone, text).
- [ ] Create each barcode symbology → it renders and scans.
- [ ] Download SVG and PNG → files save (not a new tab); a print-ready export still scans at small print size.
- [ ] Invalid input surfaces validation; a duplicate slug is rejected; the builder preview equals the downloaded asset.
